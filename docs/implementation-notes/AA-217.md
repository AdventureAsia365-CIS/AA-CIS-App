# AA-217 — Haiku retry=1 malformed-JSON empty-gen + cost-accounting leak

## Context (diagnostic, STEP 0)
CloudWatch (`/ecs/aa-cis-dev`, 2026-06-18) showed two empty-generation cells. Corrected
diagnosis overturned the original "sonnet empty-gen" framing:

- Both empty cells were **Haiku on retry=1**, not Sonnet. The Sonnet calls in the same run
  (10:15:22, 10:16:16) succeeded (scores 9.625 / 9.75).
- Root cause was **structurally malformed JSON mid-document**
  (`Expecting ',' delimiter` at char 8090 / 6528), NOT max_tokens truncation —
  `out_tokens` were 3569 / 2795, below the 4096 cap.
- The "zero cost" symptom was a **cost-accounting artifact**: the failing returns in
  `generate_node` omitted `cost_usd`, so the already-billed Bedrock spend
  (0.006044 / 0.005086) was discarded from state.
- The 06-18 cells were **pre-AA-216** (cap committed 2026-06-20 21:00). AA-216 already
  closes the score-gate hole; AA-217 is strictly the cost-leak + malformed-JSON salvage.

## Decisions
- **Deterministic salvage, no LLM re-ask.** Use `json-repair` (`repair_json(raw,
  return_objects=True)`) instead of re-prompting. Keeps generate_node single-call,
  zero extra latency/cost, fully unit-testable offline.
- **Salvage acceptance gate = `isinstance(salvaged, dict) and salvaged.get("name")`.**
  A repaired object only counts if it is a dict with a non-empty `name` (the one field
  `validate_node` keys structural completeness on). Garbage that `repair_json` turns into
  a `str` or `list`, or a dict without `name`, falls through to the fail branch → empty
  gen → AA-216 cap still blocks it.
- **`json-repair` pinned at `==0.61.0`** (exact), under the `# Utils` block in
  requirements.txt (file is grouped-by-section, not globally alphabetical).
- **No `stop_reason` logging.** `shared/llm_client/client.py` does not surface Bedrock
  `stop_reason` (line ~120 extracts `content[0]["text"]` only). Out of scope per the
  agreed cut — see Follow-up. client.py was deliberately NOT touched.

## Changed (vs. original requirement)
- Original issue framed as "sonnet empty-gen". Corrected to **Haiku retry=1 malformed
  JSON + cost leak** after log evidence. Sonnet path untouched.
- **B3 dropped.** Hypothesis was that retry feedback injection contaminates JSON output.
  Verified false: `state['feedback']` is `"; ".join(issues)` — plain validation-issue
  strings, no raw content/quotes spliced into the prompt. Nothing to harden.

## Tradeoffs
- `repair_json` is best-effort and can "fix" into a wrong-but-valid shape. Mitigated by
  the `name` acceptance gate plus the downstream AA-216 structural cap + validate/judge
  gates — a mis-salvaged object that is structurally thin still cannot reach gold.
- Salvaged content is accepted as-is (no re-validation beyond the existing pipeline). This
  is acceptable because validate_node runs next regardless and re-scores it.

## Should know (before reading the diff)
- `generate_node` (`services/content_generation/graph.py`): the standalone outer
  `except json.JSONDecodeError` was **removed**. JSON parsing is now an inner
  `try/except` around `json.loads(raw)`; the outer try keeps only `except Exception`.
- `resp = None` is initialized above the outer `try`. The inner JSONDecodeError branch
  has `resp` guaranteed bound (uses `resp.cost_usd` directly); the outer `except
  Exception` may have `resp is None` (client.generate itself raised) and is guarded with
  `(resp.cost_usd if resp else 0)`.
- Both failure returns now carry `cost_usd = state.get("cost_usd", 0) + <resp cost>` so
  billed spend is no longer dropped.
- `json_parse_failed` log enriched: `raw_len`, `char_offset` (`e.pos`), `model_used`,
  `fallback_used`, `retry_count`. New `json_repair_salvaged` info log on successful
  salvage.
- Success return dict (the happy path) is byte-for-byte unchanged.

## Tests
`tests/unit/test_aa217_malformed_json_cost.py` — 5 pure tests (LLMClient patched, no
AWS/network): salvage success, unsalvageable→empty+log+cost, JSONDecodeError cost carry
(X+Y), Exception path resp=None (no UnboundLocalError, cost=state only), fence-wrapped +
internally malformed salvage-after-strip. Full unit suite: 374 passed.
