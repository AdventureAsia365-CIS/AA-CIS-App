# AA-201 [AA-193·F5-followup] — seo_meta under-length fix

## Context
AA-194 fixed "seo_meta cut mid-word" by trimming to the last sentence-terminating
punctuation within 155 chars (`_trim_to_word_boundary(..., sentence=True)`). That
over-corrected: with no lower bound it cut to the *first* period it found, producing
78-93 char meta (S63: Atlas&Hearth 78, Trail Pulse 93, WildKind 93; only Terra 148).
AA-201 ports the Cowork v5 repair-to-band mechanism into the cloud graph.

## Decisions
- **Removed the sentence-cut at the DB write** rather than adding a floor to it. v5 only
  trims on word boundary when >155; it never cuts to a sentence. Keeping `sentence=`
  param on `_trim_to_word_boundary` (title + other callers untouched); seo_meta simply
  stops passing `sentence=True`.
- **Floor enforced in `validate_node`** (new `META_TOO_SHORT`, −0.5, dim `seo`) so a
  short meta becomes a retry/fix signal upstream, not silently shipped.
- **`META_INCOMPLETE_SENTENCE` upgraded** from "last char ∈ .!?" to `_meta_complete_sentence`:
  ends with period + last word not in `BAD_META_ENDINGS` + contains a linking/action verb.
- **flag_fix prompt** gains seo_meta repair rules only when `seo_meta ∈ fix_keys`.

## Changed vs. original requirement
- Spec PATCH 3(b) said "add META_TOO_SHORT to the FAILURE CODES list". flag_fix_node has
  **no separate valid-codes list** — the valid set *is* `STAGE2_FIX_MAPPING.keys()`, so the
  mapping entry (3a) already covers it. No second edit existed to make.
- Spec PATCH 4 gave the new seo_meta hint as one long line. It lives inside a triple-quoted
  prompt template (literal text, not a Python dict), so implicit string concat is impossible;
  split across physical lines to stay ≤120 (flake8). Adds newlines into the rendered prompt —
  harmless for the LLM.

## Tradeoffs
- Bỏ verb-whitelist (false-positive trên động từ hành trình hợp lệ: moves/winds/threads...),
  thay bằng min-word-count >=8 + BAD_META_ENDINGS. Bắt câu cụt, không bắt văn phong. Khác
  PATCH 2(b) gốc; verified trên dữ liệu kiểu S63 (3 POS / 3 NEG).
- Floor is a soft −0.5 (not a hard reject) — consistent with SEO_META_TOO_LONG weighting.

## Should know
- `validate_node(state)` reads only `state["generated"]` and `state["tour"]`; the test calls
  it directly with a baseline generated dict and asserts only on META_* codes (other noise
  codes ignored).
- `_trim_to_word_boundary` still has `sentence=` — do NOT delete it; title/other callers and
  the param signature stay. Only the seo_meta call site dropped the arg.
- No migration. Code-only. CI runs `tests/unit/` only.
