## AA-46: S4 Blog Engine

### Decisions
- **LangGraph StateGraph**: Used compile() without checkpointer (no postgres async setup needed). graph.ainvoke() called from async FastAPI handler.
- **content_calendars not found**: brief_node uses inline request data (primary_keyword, outline, title, target_keywords) — no DB fetch. When content_calendars table exists (M5), swap brief_node to fetch from DB.
- **Async LangGraph nodes**: Nodes are async functions; graph compiled without checkpointer → StateGraph.compile().ainvoke() works in async FastAPI context.
- **acp_runs INSERT**: Uses columns: run_id, tenant_id, country, status, started_at. country='S4' as proxy label.
- **Background task**: POST /runs → INSERT acp_runs (status='running') → asyncio.create_task(run_pipeline()) → return {run_id, status:'queued'} immediately. Graph runs in background.
- **Rewrite loop**: rewrite_count max 2. post_process_node violation → increment + go back to draft_node. evaluate_node gate fail → increment + go back with targeted prompt. validate_node fail → NOT auto-rewrite (goes straight to hitl_gate3_status='flagged_human').
- **SEO node**: Deterministic only — keyword density, seo_title length (50-70), meta length (150-170), keyword presence. No DataForSEO call.
- **Bedrock model**: us.anthropic.claude-sonnet-4-5-20251001-v1:0 (cross-region inference profile). AWS_REGION env var is us-west-1.

### Changed
- blog_drafts: calendar_item_id is UUID in schema but brief_node now accepts string — stored as UUID cast.
- hitl_gate3_status: 'pending' when validation_passed, 'flagged_human' when validation failed.
- run_id in POST /runs: generated UUID4 client-side → INSERT acp_runs → used for blog_drafts.run_id.

### Tradeoffs
- No checkpointer: means S4 runs can't resume on failure. Acceptable for M4.
- Inline brief data: loses calendar scheduling context. Acceptable until content_calendars table is created.
- Background task: state stored in DB only (no in-memory state). GET /runs/{run_id} polls acp_runs.status.

### Should know
- blog_drafts.calendar_item_id is NOT NULL in schema — tests must provide a valid UUID.
- graph.ainvoke() requires all state keys to be initialized in the initial state dict.
- ValidatorAgent db_rules are fetched fresh per run (not cached) — ensures rules always current.
