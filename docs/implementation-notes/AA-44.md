## AA-44: S0 Data Quality Review — Migration + API + Review Page

### Decisions
- Decision: `field_coverage_pct` computed in Python (not SQL) — easier to unit test, no extra CASE columns in query.
- Decision: GET default filter = `review_status IN ('pending_review','reviewed')`; status param overrides to single value. Approved tours visible with `?status=approved`.
- Decision: PATCH uses COALESCE to only write provided fields — safe since src_name is NOT NULL.
- Decision: date range filter added to GET (`date_from`, `date_to` on `created_at`) — UI spec has date picker but API spec didn't name params; named them conventionally.
- Decision: no EventBridge removal needed — no EventBridge references found anywhere in codebase.
- Decision: migration 024 uses VARCHAR(20) + CHECK (not enum) per spec; no type creation needed.
- Decision: admin workspace layout is separate route group `(admin)` from tenant `(tenant)` — different sidebar nav, admin branding.
- Decision: `reviewed_by` not wired in PATCH/approve/reject — no user identity available in current JWT claims (claims only have `sub` = tenant_id). Left NULL for now.
- Decision: inline editing in UI sends PATCH on Enter/blur with only the changed field populated.

### Changed
- main.py: added `v1_s0_router` import + include_router.
- Raw_tours table gains 4 nullable columns + 1 index (no data migration needed, existing rows default to 'pending_review').

### Tradeoffs
- LIMIT 500 on GET — no cursor pagination. Acceptable at current data volume (7 tours in dev). Add pagination in M3 if needed.
- `reviewed_by` always NULL until user-identity is added to JWT in a future sprint.

### Should know
- `src_highlights` is JSONB — coverage check treats any non-null JSONB value (including `[]`) as "present". This matches the spec intent (field was parsed, even if empty array).
- Reject requires non-empty notes (400 if `notes.strip()` is falsy).
- The `review_status` column on raw_tours is separate from `review_status_enum` in review_queue — different table, different type (VARCHAR vs enum).
