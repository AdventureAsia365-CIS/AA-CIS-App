## AA-43: S2 LangGraph Research Agent Scaffold

### Decisions
- Linear graph instead of conditional edges: each node checks its own condition internally (no-op if not applicable). Simpler to maintain and debug vs. branching conditional edges in LangGraph.
- Closure factory pattern (`make_*_node(pool, s3, api_keys)`) for dependency injection into LangGraph nodes — avoids global state and keeps nodes testable in isolation.
- Semaphore pre-check in router uses `_get_semaphore` from acpcore internals: direct sync check before DB insert allows synchronous 429 response. Background task then holds semaphore via `async with sem:`.
- `langgraph-checkpoint-postgres` not installed; graph.py falls back to `MemorySaver` gracefully. Production must install the package and set `DATABASE_URL`.
- `existing_content_risk` added to S2AgentState (not in original spec) — required by cannibalization check output.
- Apify node fires crawler run and stores run_id to S3 immediately (does NOT wait for crawl completion) — Apify runs are async by design; synthesis uses available data.
- pytrends wrapped in `run_in_executor` to avoid blocking the event loop (sync HTTP library).
- Reddit uses raw JSON API (no PRAW auth) for public search — avoids OAuth credential management for a scaffold.

### Changed
- Migration 028 created locally (was applied to DB already per session context but SQL file was missing).
- `acp_shared.idempotency_keys` and `acp_silver_s2.visibility_reports` both covered in migration 028.
- Router fires graph via `asyncio.create_task` with the semaphore held in the background coroutine; HTTP response returns immediately without waiting for graph completion.

### Tradeoffs
- TOCTOU race on semaphore: pre-check + background acquire has a tiny window where a 3rd request passes the pre-check before the task acquires the semaphore. Acceptable for scaffold (max 2+ε concurrent, not a security boundary).
- MemorySaver fallback loses state across pod restarts — fine for dev, broken for production recovery. Must add `langgraph-checkpoint-postgres` before production deploy.
- Apify stores only run metadata (not crawl results) to S3 since crawl is async — synthesize node won't have actual competitor content. Acceptable for scaffold; real implementation needs a polling/webhook step.

### Should know
- `acp_gold_output.published_content` table must exist for cannibalization check — if it doesn't exist in DB, dataforseo node will fail. The check is `try/except`-wrapped to degrade gracefully.
- Google Trends via pytrends is rate-limited and may fail silently — node logs warning and continues with empty data.
- Secret `aa-cis/dev/api-keys` loaded once at startup in `graph.py`'s `build_s2_graph` caller (router init); keys are passed into node factories at graph build time.
- `pytrends` and `langgraph-checkpoint-postgres` added to requirements.txt — not yet installed in current venv.
