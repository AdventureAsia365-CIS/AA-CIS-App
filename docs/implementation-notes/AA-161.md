# AA-161 — UAT Fixes Round 1 (B1–B7)

## Decisions

- B1: Already fixed in AA-159 (admin.py has FILTER WHERE source_status='active'). No change needed.
- B2/B3: Existing `/admin/master/{tour_id}/status` endpoint handles both activate/deactivate. Added two alias endpoints `/activate` and `/deactivate` that delegate to same logic with hardcoded status.
- B4: API returns `event_type` + `payload` (no title/message columns in DB). Added server-side `title`/`message` derived from event_type + payload. Frontend already maps correctly.
- B5: admin_settings.py PATCH looks correct. tenant_seo_config has `id` SERIAL PK, TEXT tenant_id — asyncpg string pass is fine. No change needed (table creation is migration 005, which is applied).
- B6: File hash check in admin_pipeline.py only checked hash, not filename. Fixed to compare cleaned filename (UUID-stripped basename) before blocking.
- B7: tours-ready did not filter source_status=trashed. Fixed + added separate trashed endpoint + frontend Trash/Restore/toggle UI.
- S1 gate: Added source_status check in `_execute_run_tour` before processing.

## Changed

- admin.py: Added `/activate` and `/deactivate` endpoints; added title+message to list_notifications
- admin_pipeline.py: Added source_status trashed gate in _execute_run_tour; fixed B6 hash-only block; fixed tours-ready exclusion; added tours-trashed endpoint
- master-content/page.tsx: Added toggleMasterStatus() + Set Active/Set Inactive buttons
- upload/page.tsx: Added Actions column + Trash/Restore + Show Trashed toggle to ToursReadySection
- AdminSidebar.tsx: Improved notification display using title/message from API

## Tradeoffs

- B2/B3 aliases: Could have just documented the existing endpoint. Added explicit aliases for clarity in future code.
- B6: UUID pattern in s3_key basename might not always be present (edge case). If stripping fails, falls back to full basename comparison — safe.
- B7: Show Trashed toggle is client-side state fetching separate endpoint. Could have been a single endpoint with filter param; chose separate for simplicity.
