# AA-235 — Tour 28-version crash: keyword_ideas shape guard

## Diagnosis (confirmed against live DB)

- **Root cause:** `silver_aa_internal.seo_context.keyword_ideas` is stored as a JSON
  **object** `{seed: null}` instead of an array `[]` when DataForSEO returns no ideas.
  The DFS seed builder appends `" tours"` to a country, so the empty result is written as
  `{"South Korea tours tours": null}`. **19 of 21 tours** have object-shaped `keyword_ideas`.
- **`_as_list` is misnamed** — 3 inline copies (`admin_pipeline.py:2291`, `:1902`, and the
  `/source` endpoint) do `json.loads(v)` and return the result *as-is*. Object string in →
  dict out. It never guaranteed a list.
- **FE crash (`"o is not iterable"`):** `DfsCompareSection` (`master-content/page.tsx:237`)
  does `const arr = [...ideas]` where `ideas = v.keyword_ideas` (typed `KeywordIdea[]`).
  Spreading a dict throws. The `useMemo` runs **before** the `if (ideas.length === 0) return null`
  guard, so it crashes unconditionally on opening Version Compare.
- **BE 500:** `export-docx` (`admin_pipeline.py:2186`) does `for kw in keyword_ideas[:25]`.
  `if keyword_ideas:` passes (non-empty dict is truthy), then `{...}[:25]` →
  `TypeError: unhashable type: 'slice'` → HTTP 500 on the DOCX button.
- **"Only the 28-version tour" = red herring.** 19/21 tours carry the bug; it only surfaces on
  heavily-rewritten tours because those are the only ones users open Compare / DOCX on. The real
  predicate is **"DFS empty → object shape"**, independent of version count. The version-detail
  and versions-list endpoints return 200 — they were *not* the 500 source. The `brand_rule_id`
  `::uuid` cast was clean (all 28 versions had valid UUIDs).

## Fix

1. **`_as_list` guarantees a list** + dedup the 3 inline copies into one helper. Object/dict,
   bad JSON, `None`, non-list parse → all return `[]`.
2. **FE `Array.isArray` guard** (defence-in-depth) at every `keyword_ideas` spread/iterate site.
3. **Backfill** the 19 object-shaped rows `{..} → []` and fix the DFS writer to persist `[]`
   (not `{seed: null}`) on an empty result, so the object shape cannot regenerate.

## Decisions

- No ADR — straight bug fix, no architectural change.
- DFS-writer fix folded into this issue (not split to AA-236) since it is the root of the
  bad data and the backfill is meaningless without it.

## Changed (vs. original requirement)

- **4 inline `_as_list` copies, not 3.** `:1810` (`/source`, param `val`) was *already*
  shape-guarded; the 3 buggy copies (`:1902` csv/xlsx, `:2056` export-docx, `:2291` detail)
  returned `json.loads` as-is. Folded all 4 → one module-level `_as_list` so the whole class
  of bug dies at one point (not just the live symptom).
- **The object data is legacy, not actively regenerated.** Current writer path
  (`fetch_all` → `_parse_keyword_ideas`) already returns a list. The `{kw:vol}` objects are
  pre-AA-203 `keywords.search_volumes` maps that an older writer stored into `keyword_ideas`.
  So fix (B) is **defence-in-depth**, not a live-bug fix.
- **Backfill = 21 rows** (full table; ~19 distinct tours — `46010051` and `a3afaed4` each had
  2 `seo_context` rows). One row `{"MOUNTAIN BIKING tours": 140}` (and a few other non-null
  values) discarded — malformed maps, unusable as `KeywordIdea[]`, accepted.
- **`handler.py:78` read-guard folded in.** Backfill side-effect: `custom_keywords` mode read
  `keyword_ideas` as a `{kw:vol}` dict; post-backfill rows are `[]`, so guard keeps
  `search_volumes` a dict (`_ki if isinstance(_ki, dict) else {}`).
- **Seed-builder bypass NOT fixed here → split to AA-236.** `build_seed()` exists
  (`seed_builder.py:62`, no double-`tours`) but `handler.py:36` uses raw `seed or destination`,
  bypassing it — root of the `"… tours tours"` / raw-name seeds. Quality bug, separate issue.

## Tradeoffs

- Keep `_as_list` as the single guard point (BE) **and** add an FE guard. Either alone fixes
  the live symptom; both gives defence-in-depth so a future non-list shape from any source
  cannot crash the modal or 500 an export.

## Should know (reviewer)

- `_as_list` behaviour change is shape-only — a valid list/array string is returned unchanged.
- Backfill is a separate, gated step (RDS running); it is destructive-ish (overwrites empty
  objects with `[]`) but only touches rows where `jsonb_typeof(keyword_ideas) = 'object'`.
  The 21 PRE-snapshot `tour_id`s above are the audit trail for the Prod data that was changed.
- Three guard layers, by design: BE `_as_list` (read), BE writer `isinstance` (write),
  FE `Array.isArray` (render). Any one fixes the symptom; all three make a non-list shape
  unable to crash the modal, 500 an export, or get re-persisted.
- Follow-up: **AA-236** = route `effective_seed` through `build_seed()` (seed quality).
