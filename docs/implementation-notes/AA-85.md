## AA-85: Tenant Detail Load Failure + Brand Rules Seed

### Decisions
- Kept two separate commits (d09a786 + 2b9ea74) rather than squashing — second crash (voice_examples) was discovered during live deploy verification, not during analysis
- Used `_parse_jsonb()` helper instead of registering asyncpg JSONB codec globally — scoped fix, no side effects on other routes
- Migration 018 applied directly to dev DB (via S3-mediated ECS exec) before code deploy; migration file committed for record but code already assumes columns exist
- Chose `COALESCE` in SQL rather than Python-side `or 0` — keeps the logic in the query where it's visible to future DB readers

### Changed
- Original requirement said "root cause = missing brand_rules row" — actual root cause was `int(NULL)` from `api_calls_quota_monthly` (no membership_plans quota row for new tenants). Brand_rules insert was still needed as prerequisite for seeding.
- Task 4 (UI overflow) required adding `display:flex + flexDirection:column + maxHeight` to outer wrapper, not just changing `overflow-hidden` as suggested — sticky header pattern needed explicit flex layout

### Tradeoffs
- `COALESCE(api_calls_quota_monthly, 0)` means new tenants always show 0% quota used, not "unlimited" — acceptable UX for now; correct fix would be inserting membership_plans rows on tenant creation
- Removed `created_at` from brand_rules SELECT — `updated_at` is sufficient for `last_updated` display; `created_at` column exists in DB but was never reliably set (NULL for older rows)
- Voice_examples stored as JSONB but asyncpg returns it as raw string — added `_parse_jsonb()` rather than configuring asyncpg codec, since only this endpoint uses the field

### Should know
- `shared.v_tenant_monthly_usage` columns (`billing_month`, `api_calls_used`, `quota_calls_pct`, `api_calls_quota_monthly`) were added directly to the DB in session P3-S5 — not captured in any migration file
- `gold_aa_internal.tenant_tour_versions` also exists but was not created by `create_tenant_schemas()` — added directly to DB at some point
- The `create_tenant` admin endpoint (POST /admin/tenants) does NOT insert into `shared.tenant_brand_rules` or `shared.membership_plans` — new tenants need manual setup for brand rules and quota display to work correctly
