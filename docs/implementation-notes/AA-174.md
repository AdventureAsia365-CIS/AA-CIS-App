# AA-174 — [OPS] Full E2E ACP run S0→S4.2 — verify UI v3 with real data

## Decisions
- Data source: `silver_aa_internal.raw_tours` country='Sri Lanka' (18 rows, review_status='pending_review').
  Selected 2 tours for the E2E run to keep S4.1 blog generation time bounded (~5 min/tour).
- Tenant: `aa_internal` (00000000-0000-0000-0000-000000000001), the default per AA-174 spec.
- S1 run trigger: `POST /acp/s1/run` via `X-Admin-Secret` header (FastAPI `_get_tenant` accepts
  admin secret and maps to aa_internal tenant). Called from inside the ECS task against
  `localhost:8000` to bypass the API Gateway Lambda Authorizer (which protects `/acp/*` and
  `/v1/*` but not `/admin/*`).

## Changed
- `api/routers/v1_s0.py`: `GET /v1/s0/review` referenced a non-existent `raw_tours.created_at`
  column in the WHERE/ORDER BY/SELECT clauses, causing every call to return HTTP 500
  (`asyncpg.exceptions.UndefinedColumnError: column "created_at" does not exist`). Renamed all
  references to `ingest_at` (the actual column), including the JSON response key
  `created_at` → `ingest_at`.

## Tradeoffs
- Bypassed the broken `/v1/s0/review` + `/v1/s0/approve` UI flow for tour selection/approval
  in this run — used a direct DB UPDATE (`review_status='approved'`) for the 2 selected
  tour_ids instead, to avoid blocking the long-running S2/S3/S4 stages on a CI/CD deploy.
  The `/v1/s0/review` fix is pushed to develop separately and will be verified once deployed.

## Should know
- No frontend currently calls `/v1/s0/review` (grep found zero references), so the JSON key
  rename (`created_at` → `ingest_at`) has no known frontend impact.
- `acp_shared.acp_runs.country` is populated from `run_config.language` (e.g. "EN-US") by
  `POST /acp/s1/run`, not the actual tour country — pre-existing behavior, not changed here.
