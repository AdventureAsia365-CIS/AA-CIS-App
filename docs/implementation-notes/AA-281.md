# AA-281 — integration test through the full compiled LangGraph StateGraph

## Decisions
- STEP 0 confirmed the gap before writing anything: `grep -rn "StateGraph\|compiled_graph\|\.compile()" tests/integration/*.py`
  returned 0 matches across all 10 existing files. Every one of them (`test_pipeline_e2e.py`,
  `test_content_generation_integration.py`, etc.) exercises pipeline stages by inserting directly
  into Postgres or calling node functions in isolation — none route through `build_graph()` +
  `.astream()`/`.invoke()`. That gap is exactly the AA-209 bug class: LangGraph silently strips any
  state key not declared in the `ContentState` TypedDict when state crosses a node boundary via the
  compiled graph — invisible to a test that calls a node function directly or seeds the DB by hand.
- Read before writing: `services/content_generation/graph.py` (full — `ContentState` TypedDict,
  `build_graph()` edges), `judge_node.py` (full — `has_brand_signals` guard, local `LLMClient`
  import), `brand_audit_node.py` (`_audit_from_judge` branch), `api/routers/v1_pipeline.py`
  (`_rewrite_tour()` — confirmed it's the real production entrypoint: builds the graph, drives it
  with `astream(stream_mode="updates")`, and its return dict uses `result.get(..., default)` for
  every judge/fallback/satellite field — meaning presence-only assertions like `"judge_score" in
  result` are always true and would be a false-positive test), `tests/integration/conftest.py`
  (`mock_llm_client` fixture pattern, `_setup_schema` autouse fixture), `tests/integration/_constants.py`
  (`SAMPLE_TOUR` shape), `shared/llm_client/models.py` (`LLMRequest`/`LLMResponse` fields).
- 3 new tests in `tests/integration/test_compiled_graph_state_propagation.py`, all driving
  `_rewrite_tour()` — never `build_graph()` or a node function directly — since that's the actual
  code path production traffic takes:
  - `test_full_graph_run_returns_judge_fields_undropped` — brand_rules includes `core_idea` so
    `judge_node`'s `has_brand_signals` guard actually lets the judge run; asserts `judge_score` /
    `judge_brand_fit` / `judge_cross_brand_distinct` / `judge_mission_present` / `quality_score`
    come out with real mocked values, not just "key present".
  - `test_full_graph_run_propagates_model_tier_metadata` — asserts `model_used`/`cost_usd`/
    `fallback_used`/`satellite_used` after a full run.
  - `test_full_graph_run_retry_path_preserves_state` — first generate attempt returns valid JSON
    missing every required field (`MISSING_FIELD` → capped quality_score → `should_retry` = retry),
    second attempt succeeds; asserts `retry_count >= 1` and the final `generated` content is the
    second attempt's, not lost across the `increment_retry` → `generate` edge.
- One mock, patched at two import sites in a single fixture (`patch_llm_client`), branching on
  `request.model_tier == "gpt-4.1"` to serve both the content-generation call and the judge call —
  see Bug (a) below for why two patches were required.
- `GOOD_GENERATED` fixture content was verified against the real `validate_node` checks
  (`SEO_META_MIN`/`MAX` from `seo_meta_utils.py`, forbidden-word lists, generic-opener/subtitle/
  highlight patterns) with a standalone script before being hard-coded, rather than guessed and
  iterated on inside the test run.

## Changed
- Added `tests/integration/test_compiled_graph_state_propagation.py` (new file). No other test

## Tradeoffs
- Two real bugs were found and fixed *while building the test*, not hypothesized — both verified
  experimentally, not from reading code alone:
  - **(a) Patch-site bug (test infra, not shipped):** `graph.py` and `judge_node.py` each do their
    own `from shared.llm_client.client import LLMClient` at module-import time and instantiate
    `LLMClient()` locally in their own namespace. Patching `shared.llm_client.client.LLMClient`
    directly is a **no-op** for both — verified by trying exactly that first: the test still
    constructed the *real* client and failed on missing AWS/OpenAI credentials instead of using
    the mock, which would have been a silent false-pass hiding the very strip bug this file exists
    to catch. Fix: patch `services.content_generation.graph.LLMClient` AND
    `services.content_generation.judge_node.LLMClient` separately (same underlying mock object,
    two bindings).
  - **(b) Fixture-data bug (test infra, not shipped):** the first draft of `GOOD_GENERATED`'s
    itinerary text contained the word "lunch". `brand_audit_node.pre_audit_checks()`'s
    `ITINERARY_MEAL_TIME_INVENTED` regex matched it, flipped `brand_audit_status` to `"flagged"`,
    which made `flag_fix_node` attempt a **real, unmocked OpenAI call** — only failed gracefully
    locally because no `OPENAI_API_KEY` is set in this shell (logged
    `flag_fix_failed_graceful`/`api_key client option must be set`). In CI, where
    `OPENAI_API_KEY=sk-test-dummy` is set, this would have been a real non-deterministic network
    call instead of a clean local failure — a landmine, not a hard failure, so easy to miss. Fixed
    by rewording the itinerary fixture to avoid meal/clock-time language; reran and confirmed the
    `flag_fix_failed_graceful` log line no longer appears (`brand_audit_status` stays `"pass"`,
    `flag_fix_node` cleanly no-ops via `_should_fix()` returning `False`).
- `tests/integration/` has no `__init__.py` (unlike `tests/unit/`, which gets repo-root
  importability transitively via `tests/__init__.py`), so `from api.routers.v1_pipeline import
  _rewrite_tour` etc. is not importable by default from this directory. Fixed locally, inside the
  new test file only, with the same `sys.path.insert(0, ...)` pattern `conftest.py` already uses
  for its own directory (to import `_constants`) — one directory level up, scoped to this file so
  no other test file's import resolution is affected.
- Did not add `docs/implementation-notes/AA-281.md`-style DB seeding or router-level
  (`POST /v1/pipeline/run`) coverage — `_rewrite_tour()` is the right unit of "full compiled graph"
  for this ticket; the HTTP layer around it is untouched by AA-209-class bugs and already has other
  coverage.
