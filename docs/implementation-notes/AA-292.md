# AA-292 — "View content" action in Rewrite History

## Decisions
- STEP 0 discovered the ticket's premise was already half-wrong: clicking a History
  table row already calls `loadVersion(h.version_num)` → `GET
  /admin/tours/{tour_id}/versions/{version_num}` → renders a full content preview
  (name/subtitle/summary/itineraries/seo_title/seo_meta/highlights) below the table,
  with its own "Make Current" button. This has existed since commit `4004d91`
  (AA-129) — confirmed via `git log -S"loadVersion"` — not something this session
  added. The row itself just had no visible affordance indicating it's clickable.
- Given that, the correct minimal-scope fix is exactly what the ticket's
  Implementation step 1 asked for taken literally: add an explicit "View content"
  button next to "Make Current" in each row — not a new fetch, not a new render
  path. The button calls the same already-existing `loadVersion()`, so this is
  pure UI discoverability, zero new logic, zero duplication.
- Did **not** reuse `CompareModal.tsx` (`frontend/app/admin/_components/CompareModal.tsx`,
  274 lines) per STEP 0 evaluation: it's built for comparing N *different tours*'
  current/latest generated content side-by-side via `GET /admin/tours/{id}/detail`
  — it has no concept of a specific historical `version_num` and always fetches
  the latest version. Adapting it to show one historical version of one tour would
  mean reworking its fetch endpoint, dropping the N-column grid layout for N=1, and
  wiring a new prop — more invasive than reusing the inline preview pattern that
  already does the job with the correct endpoint.

## Changed
- `frontend/app/admin/_components/TourDetailPanelV2.tsx`: in the History tab table,
  each row's action cell now has two buttons ("View content", "Make Current")
  instead of one. "View content" calls the pre-existing `loadVersion(h.version_num)`.
  No other code touched — `loadVersion`, `selectedVersion`, the preview render
  block, and `promoteVersion` are all unchanged.

## Tradeoffs
- The row remains clickable too (pre-existing `onClick={() => loadVersion(...)}`
  on `<tr>`) — left as-is rather than removed, since removing it would be a
  behavior change beyond "chỉ thêm, không sửa/xoá gì có sẵn" (task's explicit
  instruction). Both the row click and the new button now do the same thing;
  this is intentional redundancy for discoverability, not a bug.
- `VersionContent` (the existing preview data shape) has no `aa_description`
  field — the preview shows name/subtitle/summary/itineraries/seo_title/seo_meta/
  highlights, not a separate "description". Left as-is since extending the
  preview's field set is a different, larger change than this ticket scoped.
