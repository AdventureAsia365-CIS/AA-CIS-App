  (dated after the original task brief) moved writer selection to a tiered policy — T1
  acc2-native preferred once it clears a quality bar, T2 Claude-acc1-satellite is now real
  billed money (Activate credits application rejected 22/07/2026), not subsidized. AA-308
  Phần A bake-off already ran (Palmyra 14/20 vs Claude 6/20, Nova blind-judged, avg 8.50 vs
  8.10) — that's the evidence gate this task was waiting on; AA-300 (curation UI) shipping
  closes the other precondition. Model id verified live: `us.writer.palmyra-x5-v1:0`,
  ACTIVE inference profile on acc2 (005097885195), no cross-account AssumeRole needed —
  `aa-cis-dev-ecs-task-role`'s existing `bedrock:InvokeModel` (Resource `*`) already covers
  it. Response shape verified live via ECS exec smoke test: OpenAI-compatible
  `choices[0].message.content` (Writer/Palmyra format), NOT Anthropic's `content[0].text` —
  confirms the docstring warning already in `bedrock_satellite.py`.
- **Writer is implemented behind one seam function** (`generate_draft(system_prompt,
  user_prompt, model_tier=...)`) so swapping Palmyra→Claude later is a one-line change, not
  a refactor. If Palmyra's step-7 verify output looks bad, that gets reported back before any
  switch to acc1 Claude — that's real money and needs separate sign-off, not something to
  flip unilaterally.
- **AA-263/ADR-2026-020 excluded from this PR.** Read the full ADR-2026-020 page (Notion):
  it explicitly targets the OLD S1 graph (`generate→validate→llm_judge→brand_audit→
  flag_fix→revalidate`, `services/content_generation/graph.py`) — "S1 hiện gộp việc kiểm
  tra vào validate_node... KHÔNG có node editor viết-lại riêng." Not implemented anywhere
  (zero code hits for AA-263). Per instruction, old S1's `graph.py`/`validate_node` is not
  touched under any interpretation in this PR — this is a separate, unrelated architecture
  change to flag for Nghiep/Ms. Thư to decide and track as its own issue, not bundled here.

## Tradeoffs

- No LangGraph for this node — plain functions with a bounded retry loop. The task's
  step 8 reference to "AA-281's compiled StateGraph pattern" doesn't exist in this repo
  (grepped, zero hits) and the control flow here (fetch atoms → draft → gate → retry ≤2)
  doesn't need conditional-edge routing to justify the dependency.
- `check_grounding()`'s citation regex (`\[R:([^\]]+)\]`) intentionally matches ANY
  token after `[R:`, not just well-formed `atom_[0-9a-f]{10}` — a hallucinated or
  malformed citation must surface as an `unknown_citations` gate failure, not be
  silently ignored because it didn't match a strict shape. Caught this via a failing
  unit test before it reached real verification.
- Density gate only checks `aa_subtitle/aa_summary/aa_highlights/aa_itineraries`, not
  `seo_title`/`seo_meta` (old S1 has both) — this task's contract only asked for the
  `aa_*` fields; SEO fields are out of scope, not silently dropped.
- Palmyra calls are sequential with manual spacing in the verify script after hitting
  `ThrottlingException` on back-to-back calls (see Should know) — no backoff/retry was
  added inside `_call_palmyra` itself since this repo's convention (v1_atoms.py's own
  comment on acc1 satellite) is "don't add retry/backoff before the rate limit is
  actually measured" — same principle applied here for acc2 Palmyra since this is the
  first real caller of it in this repo.

## Should know

- **AA-263/ADR-2026-020 explicitly out of scope, confirmed not ambiguous.** Read the
  full ADR-2026-020 Notion page: it names old S1's 7-node graph
  (`generate→validate→llm_judge→brand_audit→flag_fix→revalidate`) directly and says S1
  currently folds the "editor" (rewrite) responsibility into `validate_node`
  (scoring) instead of keeping it as its own node, per aa-marketing-v2 research.
  This targets `services/content_generation/graph.py`, which this PR does not touch.
  Not implemented anywhere in the repo. Flagging for Nghiep to track as its own issue
  if still wanted — did not bundle it here per task instruction.
- **Palmyra X5 on acc2 throttles under back-to-back sequential calls with no spacing.**
  First live verify run (4 tours, no delay between calls) got 1 success + 3
  `ThrottlingException` (botocore's built-in retry exhausted at 4 attempts each). A
  second pass with 25s between calls got 2 more through; the last needed one more
  standalone retry with a longer gap. All 4 eventually succeeded with real content
  and a clean gate pass. This is real, measured acc2 Palmyra throughput behavior
  worth knowing before wiring this into any batch/multi-tour flow — no RPM/TPM number
  is published anywhere I found, only this empirical throttle-and-back-off behavior.
- **Real live verify results, 4/4 curated tours, model_tier="palmyra", 0 retries needed
  once spaced correctly:**

  | Tour | atoms avail. | atoms used | citations | words | words/citation | gate |
  |---|---|---|---|---|---|---|
  | Classic Exploration (`c410a272`) | 21 | 20 | 38 | 570 | 15.0 | pass |
  | Classic Exploration (`cc2eca43`) | 18 | 18 | 39 | 533 | 13.7 | pass |
  | A sea of coral... (`a3afaed4`) | 6 | 6 | 21 | 382 | 18.2 | pass |
  | Yaksa Trek BEST DEAL (`296abe67`, thin) | 2 | 2 | 7 | 136 | 19.4 | pass |

  All well under the 300-words-per-citation gate; zero `unknown_citations` (no
  hallucinated atom_id) across all 4. The thin-trip tour (2 atoms) produced a short,
  honest 136-word page instead of being padded to a target length — exactly the L1/D3
  intent.
- **Glass-box comparison (`c410a272`, Classic Exploration) — read both outputs, don't
  just trust the density number.** Old S1's `aa_summary` includes at least one
  generalization not obviously traceable to a single fact ("mornings tend to anchor
  the day's primary activity, leaving afternoons for transit and acclimatization") —
  plausible-sounding but the kind of model-prior padding ADR-2026-024 names as the
  core problem. S1-from-atom's equivalent fields cite a real atom for every concrete
  claim instead.
  **Real limitation observed, not fixed in this PR:** `aa_summary` and
  `aa_itineraries` draw from the same finite atom pool and substantially overlap —
  e.g. the Negombo Fishing Village atom is restated almost identically in both the
  summary and Day 1 of the itinerary, across all 4 test tours. Old S1's summary reads
  as a distinct higher-level "essay" layer; the grounded version doesn't have that
  separation because both fields are independently assembling from the same atoms
  with no instruction to differentiate scope. Flagging for Nghiep — this task's scope
  was grounding/gating/writer selection, not itinerary-vs-summary content shape.
