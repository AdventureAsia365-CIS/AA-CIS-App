## AA-276 (Bug 3): export 500 (tour mới) — `metadata.brand_rule_id` empty-string cast crash

### Decisions
- Guard tại SQL layer (`NULLIF(gc.metadata->>'brand_rule_id', '')::uuid`) ở CẢ 3 route đụng
  pattern này, không chỉ 2 route được liệt kê ban đầu — tìm thấy 1 chỗ thứ 3
  (`GET /tours/{tour_id}/versions/export`, dòng ~2256) có cast y hệt, cùng lỗi tiềm ẩn.
  Không sửa thì đây là 1 known-vulnerable pattern còn sót lại trong cùng file.
- Root cause fix dùng `Optional[str] = None` thay vì `""` cho `brand_rule_id` — xác nhận
  bằng cách đọc code serialize thật (`json.dumps(_build_generated_metadata(...), default=str)`):
  `None` → JSON `null` tự nhiên (built-in, không qua `default=str` fallback) → lưu vào
  jsonb như SQL NULL thật → `metadata->>'brand_rule_id'` trả NULL, không phải `""`.
  Test `test_root_cause_fix_serializes_missing_brand_rule_id_as_json_null_not_empty_string`
  khóa hành vi này lại bằng round-trip json.dumps/json.loads thật, không giả định.
- Backfill (Phần 4) dùng `jsonb_set(metadata, '{brand_rule_id}', 'null'::jsonb)` — giữ key
  tồn tại với value JSON null, khớp đúng với cách Phần 2 sẽ serialize cho các row MỚI (thay
  vì xoá hẳn key bằng `metadata - 'brand_rule_id'`, vốn cũng hoạt động đúng về mặt
  `->>'brand_rule_id'` nhưng không nhất quán shape với dữ liệu mới).

### Changed
- `api/routers/admin_pipeline.py`:
  - dòng ~2256 (`export_tour_versions`), ~2385 (`export_tour_version_docx`), ~2606
    (`get_tour_version_detail`): `(gc.metadata->>'brand_rule_id')::uuid` →
    `NULLIF(gc.metadata->>'brand_rule_id', '')::uuid`.
  - dòng ~343/354 (`_execute_run_tour`): `brand_rule_id: str = ""` →
    `Optional[str] = None`; else-branch khi `_resolve_brand_rule()` không tìm thấy row
    cũng đổi `else ""` → `else None`.
- `tests/unit/test_aa276_brand_rule_id_null_guard.py` (mới): 4 test — 3 test gọi qua
  endpoint thật (không phải helper riêng lẻ) với metadata mang đúng giá trị lỗi thật
  (`brand_rule_id: ""`) từng gây 500 trong prod, assert SQL text có NULLIF guard VÀ
  response không crash; 1 test pure-Python khóa hành vi serialize None→null.

### Tradeoffs
- Guard SQL (Phần 1) là lưới an toàn cho MỌI giá trị rác tương lai (không chỉ `""`) —
  chấp nhận việc nó "che" một class lỗi rộng hơn root cause cụ thể này, đổi lại endpoint
  không bao giờ 500 vì 1 cast lỗi ở cột metadata JSONB tự do.
- Không sửa `_resolve_brand_rule()` — xem Phần 3, không tìm thấy lỗi cụ thể để sửa.
- Regression test dùng fake pool/conn (không phải DB thật) theo đúng convention sẵn có
  của `test_aa219_version_detail.py` / `test_aa220_export.py` — do đó test KHÔNG tự chạy
  Postgres cast thật; nó khóa (a) SQL text phải chứa NULLIF guard, và (b) endpoint code
  Python không crash khi nhận lại 1 row đã được guard đúng. Test không thể tự phát hiện
  nếu ai đó xoá NULLIF nhưng giữ nguyên literal string khác — chấp nhận được vì đây vẫn
  là cách duy nhất test unit (không cần Postgres/AWS) có thể pin hành vi này.

### Phần 4 — Backfill SQL cho 1 row dữ liệu rác hiện có (CHƯA CHẠY — chờ go signal)

```sql
-- STEP 1 — Pre-check: xác nhận đúng 1 row bị ảnh hưởng trước khi UPDATE.
SELECT id, tour_id, version_num, metadata->>'brand_rule_id' AS brand_rule_id_raw
FROM silver_aa_internal.generated_content
WHERE tour_id = '7f57ba60-7fa5-4842-918f-79e823c6f380'::uuid
  AND version_num = 1
  AND metadata->>'brand_rule_id' = '';
-- Kỳ vọng: đúng 1 row. Nếu count != 1, DỪNG — không chạy STEP 2.

-- STEP 2 — Backfill: chuẩn hoá brand_rule_id="" thành JSON null (khớp cách Phần 2
-- serialize cho row mới: json.dumps(None) -> "brand_rule_id": null).
UPDATE silver_aa_internal.generated_content
SET metadata = jsonb_set(metadata, '{brand_rule_id}', 'null'::jsonb)
WHERE tour_id = '7f57ba60-7fa5-4842-918f-79e823c6f380'::uuid
  AND version_num = 1
  AND metadata->>'brand_rule_id' = '';
-- WHERE scoped theo tour_id + version_num + đúng giá trị lỗi — chỉ có thể đụng
-- đúng 1 row đã biết (hoặc 0 row nếu đã fix / đã chạy trước đó).

-- STEP 3 — Verify sau khi chạy STEP 2.
SELECT id, tour_id, version_num, metadata->>'brand_rule_id' AS brand_rule_id_raw
FROM silver_aa_internal.generated_content
WHERE tour_id = '7f57ba60-7fa5-4842-918f-79e823c6f380'::uuid AND version_num = 1;
-- Kỳ vọng: brand_rule_id_raw IS NULL (không phải '').
```
