# AA-204 [F5-over-band] — over-length seo_meta repair + observability + sentence backstop

## Context (S64 diagnostic)
`validate_node` fired `SEO_META_TOO_LONG` into `state["failure_codes"]`, but `flag_fix_node`
built `fix_keys` only from `state["brand_audit_codes"]` (gated on `brand_audit_status=="flagged"`),
and `SEO_META_TOO_LONG` was absent from `STAGE2_FIX_MAPPING`. So over-length meta never reached the
graceful in-graph repair — it was only blunt-trimmed on the DB-write path (`sentence=False`),
landing in-length but mid-sentence (DB rows ended on `d`/`s`/`g`). The API return surfaced the raw
untrimmed value (165/186), diverging from DB.

## Decisions
- **Extracted `_should_fix(state)` and `_build_fix_keys(state)` as pure helpers** in flag_fix_node.
  `flag_fix_node` invokes the real LLMClient, so the gate + key-selection logic could not be unit
  tested inline. Helpers keep the AA-204 test free of I/O (mirrors test_aa201 style). Reported to user.
- **`_DETERMINISTIC_SEO_CODES = {SEO_META_TOO_LONG, META_TOO_SHORT, META_INCOMPLETE_SENTENCE}`** at
  module level — these validate-layer codes are deterministic, so they drive repair regardless of the
  (non-deterministic) brand-audit status.
- **Backstop kept inline** (3 lines) per brief, not extracted — tested via `_trim_to_word_boundary`
  both modes + replicating the selection rule in the test.

## Changed
- **flag_fix_node.py**: `STAGE2_FIX_MAPPING["SEO_META_TOO_LONG"]="seo_meta"`; new `_should_fix` /
  `_build_fix_keys` helpers + `_DETERMINISTIC_SEO_CODES`; node body uses them; gate now runs when
  flagged OR a deterministic SEO code fired (clean pass still skips); meta_rules prompt gains an
  over-length SHORTEN rule + `Current SEO_META (N chars): ...` context line.
- **admin_pipeline.py**: DB-write trim is now sentence-aware with a 140 floor fallback —
  `_m_sent = trim(…,155,sentence=True); _m_final = _m_sent if len>=140 else trim(…,155)`; INSERT uses
  `_m_final`. Return dict surfaces `failure_codes` and `seo_meta_stored` (=`_m_final`) for observability.
  `_m_final` initialized to `None` before the generated-guard so the return is always safe.
- **graph.py / brand_audit_node.py**: NO change. Verified `failure_codes` carries through to flag_fix
  (declared in ContentState; every brand_audit return uses `{**state, ...}`).
- **tests/unit/test_aa204_meta_over_band.py**: 7 pure tests (mapping, routing when audit=pass,
  short/incomplete routing, clean-pass skip, flagged merge, sentence-aware trim in band, floor fallback).

## Tradeoffs
- Backstop sentence-trim could dip under 140 (the original AA-201 bug); the floor fallback to the
  longer word-boundary cut prevents that. Repair (flag_fix) is the primary fix; backstop rarely fires.
- `generated` kept raw in the return for backward-compat; `seo_meta_stored` added alongside rather than
  mutating `generated`.

## Should know
- Two independent meta mechanisms now reconciled: validate flags → flag_fix repairs (deterministic
  route) → admin trim is only the final net. The over-length code finally has a path into repair.
- No migration. No new env/secret.
