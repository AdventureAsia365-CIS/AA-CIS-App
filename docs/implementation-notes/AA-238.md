After AA-231 forced Sonnet to reach floor 140, the model pads length with "public transport" / "public-transport" → STRICT RULE 7 forbidden word → brand audit → HITL. 2/3 Sonnet runs fall HITL for a *new* reason (forbidden word), not length.

## Root cause (STEP 0 confirmed — read-only)
- Band guard (`seo_meta_utils.meta_in_band`) checks **length 140–155 + `meta_complete_sentence` only**. No forbidden-word criterion anywhere in `meta_in_band` / `best_meta_candidate` / `_rerepair_meta`. A pad that reintroduces budget language is accepted as "in band".
- Only post-fix forbidden re-check is `validate_node` **score deductions, not a hard gate**:
  - `BRAND_SEO_META_VIOLATION` (graph.py:359-367) = −1.5, break on first hit
  - generic `FORBIDDEN_WORD` (graph.py:341-345) = −0.5 each
  A single budget-word pad (−1.5) on otherwise-strong content can stay **above the 7.0 gate → silent to gold without HITL**.
- Three forbidden sources diverge: validate `_seo_meta_forbidden` = {hostel, budget, public transport, cheap, backpacker, dorm} vs general forbidden list vs tenant `brand_forbidden_words`. `_rerepair_meta` prompt mentions **none** → re-repair LLM can re-introduce a banned word while "fixing" length.

## Decisions
- **D1 — forbidden-free as HARD criterion of band guard.** A meta containing a forbidden word is NEVER "in band" → forced into repair/escalation, cannot pass on score alone. Closes silent-to-gold deterministically.
- **D2 — NO deterministic word substitution.** Brand wording is Ms. Thư's. Reject candidate + inject the forbidden list into `_rerepair_meta` prompt ("avoid: hostel/budget/public transport/cheap/backpacker/dorm") so the LLM rewrites around it. If re-repair still can't avoid → `_meta_floor_failed=True` → `manual_check`/HITL (correct outcome).
- **D4 — unify forbidden list** = `_seo_meta_forbidden ∪ tenant brand_forbidden_words`, built once per node, passed down. Code-structure only, no new brand word seeded → no Ms. Thư content sign-off. May change HITL volume → flag in close note (non-blocking).

## Fix design
- `seo_meta_utils.py`
  - add `meta_has_forbidden(meta, forbidden) -> bool` — normalized match: lowercase, NFKD accent-strip (per AA-115), treat "public-transport" == "public transport" (hyphen/space fold), word-boundary.
  - `meta_in_band(meta, forbidden=None)` — when `forbidden` provided, in-band requires `not meta_has_forbidden`. `forbidden=None` keeps legacy behavior (other callers unaffected).
  - `best_meta_candidate(post, pre, forbidden=None)` — thread `forbidden` into every in-band test so a forbidden candidate is never returned as in-band.
- `flag_fix_node.py` (repair chain 208-232)
  - build `meta_forbidden = _seo_meta_forbidden | set(tenant brand_forbidden_words)`; pass to `best_meta_candidate` + `meta_in_band` + `_rerepair_meta` clue.
  - existing escalation (`_meta_floor_failed → brand_audit_status="manual_check"`) now also fires on un-fixable forbidden — intended.
- `validate_node` scoring unchanged (defense in depth, secondary signal).

## STEP 2 — Actual implementation (supersedes STEP 1 scope)

**Scope corrected: 2 files → 3 files.** STEP 1 design assumed seo_meta_utils.py + flag_fix_node.py. Reading the real code revealed an architecture gap forcing a graph.py change:

- `BRAND_SEO_META_VIOLATION` (validate_node, −1.5 score) was NOT in `_DETERMINISTIC_SEO_CODES`. So `_should_fix` returned True only when brand_audit (LLM, non-deterministic) flagged. A forbidden-word meta that brand_audit missed → flag_fix SKIPPED → the new hard-gate never ran → forbidden meta could still reach gold on score alone. Fix requires routing the deterministic code into the repair pass.
- **Decision: reuse existing `BRAND_SEO_META_VIOLATION`** (already fires on forbidden-in-seo_meta), NOT a new `META_FORBIDDEN_WORD`. Added it to `STAGE2_FIX_MAPPING` (→seo_meta) + `_DETERMINISTIC_SEO_CODES`. Less churn, no new string.
- **Hyphen bug (root of v11 non-determinism):** validate's `"public transport" in seo_meta_lower` did NOT catch "public-transport". Added `.replace("-"," ")` in graph.validate + `_normalize_meta` (lowercase + NFKD accent-strip + hyphen→space fold) in seo_meta_utils. This is what made v11 deterministic.
- `best_meta_candidate` kept return type `str` (NOT Optional) to preserve the AA-205 caller contract; forbidden/un-salvageable signalled via not-in-band so flag_fix escalates to `_rerepair_meta` → `_meta_floor_failed` → manual_check.
