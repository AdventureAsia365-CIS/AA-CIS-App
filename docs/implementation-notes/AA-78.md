## AA-78: Brand Brief Parser Lambda

### Decisions
- Flatten DOCX paragraphs to individual lines before parsing (split on `\n` within paragraphs), because some docs pack "Brand type:" + "Core idea:" in one paragraph with soft breaks, while others split them across paragraphs
- `_is_section_header()` tested first on each line so section collection always terminates at the right boundary
- `customer_segment` collected as multi-line (some docs have 2 content lines)
- `forbidden_words` = quoted phrases extracted with regex + unquoted key terms from "Do not use X language" patterns (covers the "backpacker" case in Atlas where no quotes used)
- `confidence` = sections_found / 8 (8 expected sections: brand_type, core_idea, markets, segment, mindset, tone, style, should_not_write)
- handler uses psycopg2 (sync) not asyncpg — Lambda sync, not FastAPI async
- FastAPI endpoint added to admin.py (not a new router) to stay consistent with the existing auth guard

### Changed
- `good_example` collection: only takes first non-header line (spec says "next paragraph") — avoids accidentally consuming Should Write content if spacing is irregular
- `customer_segment` joined with newline when multi-line, not comma — preserves original structure for readability

### Tradeoffs
- psycopg2 in Lambda vs asyncpg: Lambda is synchronous; asyncpg would require an event loop wrapper. psycopg2-binary avoids that complexity.
- Confidence threshold 0.6 (3 out of 8) = below threshold aborts DB write — generous enough to not fail on a reformatted brief that's missing one or two minor sections
- `forbidden_words` lowercased for dedup check but original case stored — tests use `.lower()` in assertions so this is fine
