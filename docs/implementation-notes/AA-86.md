## AA-86: Structured failure codes + sub-scores in validate_node

### Decisions
- Decision: keep the flat `score` accumulation variable (all existing `score -= X` lines preserved) but do NOT return it; final quality_score = average of 4 sub-scores computed from `fired` list. This avoids removing any existing logic while making the DB writes accurate.
- Decision: `failure_codes` stored as unique list (dict.fromkeys preserves first-seen order, deduplicates). `failed_count = len(failure_codes)`, `passed_count = dims where sub_score == 10.0`.
- Decision: `ITINERARY_STRUCTURE_WEAK` fires if either: no day-marker regex match OR itinerary len < 80. Empty itinerary hits both — this double-codes with MISSING_FIELD but sub_scores handle the floor via max(0.0,...).
- Decision: `DFS_INTENT_UNDERUSED` skips silently when seo dict is empty (no SEO run). Guard: `if seo_kws`.
- Decision: validator_fn_version bumped from 'v1' to 'v2' in quality_scores INSERT to signal schema change.

### Changed
- ContentState TypedDict: added failure_codes, sub_scores, passed_count, failed_count.
- validate_node return dict: new fields alongside existing quality_score + feedback.
- _rewrite_tour return dict: passes failure_codes, sub_scores, passed_count, failed_count through.
- quality_scores INSERT: $4-$11 now individual sub-scores + failure_codes + counts (was $4 copied 4 times, counts hardcoded 0).

### Tradeoffs
- Sub-score average ≠ old flat score numerically. Tour previously scoring 8.5 may now score 9.1 (if only 1 dim penalised). Threshold 7.0 is unchanged; a few borderline tours may shift.
- highlights_text join guarded with isinstance check — defensive fix for pre-existing edge case.

### Should know
- MISSING_FIELD fires once per missing field — fired list may have duplicates. Sub-score sum accumulates all deductions correctly. failed_count uses unique codes so it won't over-count.
- The 3 new checks (META_INCOMPLETE_SENTENCE, ITINERARY_STRUCTURE_WEAK, DFS_INTENT_UNDERUSED) add up to 3.0 max additional penalty. Tours with short/unstructured itineraries will drop into HITL more often.
- Test `test_validate_perfect_content` still gets 10.0: perfect fixture has day markers, seo_meta ends with '.', no seo keywords to check.
