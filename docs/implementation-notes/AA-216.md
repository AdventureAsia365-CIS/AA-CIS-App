# AA-216 — Empty-gen gate hardening

Cap `quality_score` when generation is structurally incomplete so empty / missing-field
content can never clear the 7.0 gate into gold.

## Decisions
- New module constant `MISSING_FIELD_CAP = 4.0` next to `MIN_QUALITY`. Mirrors the judge's
  `_MISSION_ABSENT_CAP` pattern (structural failure caps the score regardless of other dims).
- `_structural_fail = ("MISSING_FIELD" in fired) or (not generated.get("name"))`. The empty-name
  check is belt-and-suspenders: a totally empty `generated` fires MISSING_FIELD anyway, but keying
  on `name` makes the intent explicit and guards a generation that somehow omits only the name.
- Cap applied to the final averaged `quality_score` via `min(quality_score, MISSING_FIELD_CAP)`.

## Changed
- `services/content_generation/graph.py`: added `MISSING_FIELD_CAP`; cap logic immediately before
  `quality_score = sum(sub_scores.values()) / 4` is consumed (computed then min'd).

## Tradeoffs
- Did NOT touch `_FAILURE_MAP` weight for MISSING_FIELD. Bumping the per-code deduction would only
  affect the `structure` sub-bucket; the 4-bucket average still can't drop empty content (brand/
  quality = 10, nothing to violate) under 7. A hard cap on the average is the only reliable lever.
- Cap value 4.0 is well under the 7.0 gate with margin, so any single legit deduction stacked on a
  structural fail still stays blocked.

## Should know
- Flow this protects: `should_retry` returns "done" only when `score >= MIN_QUALITY`; capping at 4.0
  forces "retry" (retry_count < MAX_RETRIES-1) then "hitl". `_is_publishable` (AA-211) also blocks.
- Pure unit tests in `tests/unit/test_aa216_empty_gen_gate.py` — no LLM/DB; call `validate_node` +
  `should_retry` on hand-built state. 5 tests. Clean full-content case asserts no regression.
