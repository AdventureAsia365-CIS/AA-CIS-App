# AA-239 — seo_meta band guard yields in-band length but cut sentence (no period)

**Status:** STEP 1 design · branch `feature/aa-238-239-seo-meta-band-guard` · live baseline api:332
**Related:** AA-231 (band enforcement), AA-205 (deterministic post-repair band guard — this path), AA-204 (over-length cut-sentence)

## Symptom (tour c9fb02ef v11, S80)
length=146 (∈ band) but cut mid-clause, no terminating period:
> "This twelve-day public-transport journey links Seoul's cultural sites with Seoraksan forest trails, an overnight temple stay in Gyeongju, and Jeju"

Length passes, sentence-completeness fails → violates STRICT RULE 8 / band guard.

## Root cause (STEP 0 confirmed — read-only)
- `_trim_to_sentence` (seo_meta_utils.py:39-52) trims **DOWN only**, to the **last** `.`/`!`/`?` within 155. It is last-period-only + downward-only.
- `best_meta_candidate` (≈67-68): post-if-in-band → else pre-trimmed-to-≤155-if-in-band → else **post unchanged**.
- Failure: an over-155 meta whose only/last interior terminator sits early (e.g. char ~90) → `_trim_to_sentence` yields ~90 chars → `meta_in_band(trimmed)` False (under 140) → discarded → `best_meta_candidate` returns **`post` unchanged** = the raw cut, no-period meta. The period-aware salvage silently no-ops whenever the interior period falls below the 140 floor. No "nearest boundary that keeps ≥140" logic.

## Decision
- **D3 — salvage upgrade.** Instead of last-period-only, enumerate sentence-boundary cut points (`.!?`) within ≤155 and pick the **largest prefix that is ≥140 AND a complete sentence**. If none keeps ≥140 → return None (cannot salvage) so the caller escalates to `_rerepair_meta` rather than emitting raw `post`. Never emit a cut/no-period meta to gold.

## Fix design
- `seo_meta_utils.py`
  - `_trim_to_sentence` (or a new `_salvage_to_band`): return the longest complete-sentence prefix ∈ [140,155]; `None` if impossible.
  - `best_meta_candidate`: on salvage `None`, do NOT fall back to raw `post` — signal not-in-band so `flag_fix` pushes `_rerepair_meta`.
- `flag_fix_node.py`: unchanged escalation — if post-`_rerepair_meta` still not in band → `_meta_floor_failed=True` → `manual_check`. (Shares branch with AA-238; same functions touched.)

## Unit repro (no AWS — regression test, tests/unit/)
- over-155 meta, only interior period at ~char 90 → old `best_meta_candidate` returns raw cut `post`; new returns None/escalates (no raw no-period meta emitted).
- over-155 meta with a complete sentence boundary at ~char 148 → salvages to that boundary, in-band + ends with period.
- boundary: exactly-140 complete sentence accepted; 139 rejected.

## Acceptance
No version emits a meta failing `meta_complete_sentence`; un-salvageable cases go HITL (not raw cut to gold). flake8 120, tests/unit/, no Co-Authored-By.
