## AA-88: Competitor URL management

### Decisions
- Decision: migration 027 (gap 020-026 reserved for other tickets, following existing pattern).
- Decision: `shared.tenants(tenant_id)` not `(id)` — spec had wrong column name, `tenant_id` is the PK.
- Decision: `added_by UUID` has no FK — `shared.users` table does not exist. Plain UUID, nullable.
- Decision: `CREATE SCHEMA IF NOT EXISTS acp_silver_s2` in migration head — schema is new.
- Decision: PATCH uses `COALESCE($label, label)` to avoid dynamic SQL building — simpler + injection-safe.
- Decision: DELETE is soft-delete only (is_active=false) as specified. Status 204 No Content.
- Decision: max-10 counts only `is_active=true` rows — deactivated URLs don't consume the slot.
- Decision: frontend path is `src/app/(tenant)/portal/...` (ACP App uses src/ convention).
- Decision: frontend stores JWT in localStorage keyed `acp_token`. If absent, shows API key input form that calls POST /auth/tenant-login to exchange key → JWT.
- Decision: portal sidebar created as part of `(tenant)/portal/layout.tsx` since no existing layout exists.

### Changed
- Migration: fixed FK to `shared.tenants(tenant_id)` and dropped `added_by` FK.
- main.py: added `v1_competitors_router` import + include_router.

### Tradeoffs
- localStorage JWT: simple, no server session needed. Risk: XSS. Acceptable for ACP B2B tool (not consumer-facing).
- COALESCE PATCH: always writes both fields even if only one changes. Negligible cost.

### Should know
- The `(tenant)` Next.js route group doesn't add a URL segment — `/portal/competitors` is the actual path.
- Sidebar is minimal: only Competitors link. Other pages (dashboard, brand identity) will be added in future sprints.
- The max-10 check happens before INSERT, inside the same pool connection (not a transaction) — low risk of race condition at current volume.
