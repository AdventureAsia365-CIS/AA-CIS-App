# AA-251 — DFS_INTENT_UNDERUSED false-positive (hướng 4: seed cụ thể + fuzzy match)

## Decisions
- `build_seed()` gains a 3rd optional arg `tour_name` (default `""`) instead of a new function —
  keeps the single seed-construction entry point, backward compatible (existing 2-arg callers
  unaffected, `test_build_seed_country_only` etc. still pass unchanged).
- Priority order: activity+country (unchanged AA-197 behavior, most specific when data exists)
  > tour_name (+ country if known) > country-only `"{country} tours"` fallback (last resort,
  the exact case that made 85% of KR/LK catalogue false-positive).
- Per-tour specific data source confirmed: `silver_aa_internal.raw_tours.src_name` — always
  populated (ingestion requires it), unlike `activities` which is frequently empty/dirty for the
  affected KR/LK batch. `admin_pipeline.py:413` already has the full `row` (`SELECT *`), so wiring
  `row.get("src_name")` through is a 1-line call-site change, no new query.
- Match algorithm: token-set overlap ratio (≥0.5 of the keyword's significant tokens must appear
  in seo_title+seo_meta) replaces exact substring-of-full-phrase. Stopword list strips generic
  tour-marketing filler ("tours", "tour", "travel", "best", "top", ...) so a keyword like
  "south korea tours" is judged on {south, korea}, not held hostage to the literal word "tours"
  appearing adjacent to the destination name.
- No fuzzy-matching library (rapidfuzz/thefuzz) added — plain token-set overlap is sufficient for
  the stated failure mode and keeps the dependency surface unchanged.

## Changed
- `services/seo_intelligence/seed_builder.py`: `build_seed()` signature +`tour_name` param.
- `api/routers/admin_pipeline.py:413`: pass `row.get("src_name")` as the new 3rd arg.
- `services/content_generation/graph.py:446-462`: substring check → `_keyword_intent_matched()`
  token-overlap helper (module-level, unit-testable in isolation).

## Tradeoffs
- 0.5 overlap threshold chosen over exact-match (too strict, caused this bug) and "any single
  token" (too loose — a keyword like "best korea beach tours" would match on "korea" alone even
  if the content is about hiking, not beaches). 0.5 requires roughly half the keyword's meaning
  to show up, which still allows a synonym or word-order difference through.
- Did not fetch DataForSEO per-tour-activity live data (would need a real API call per tour) —
  out of scope per ADR-2026-021 hướng 4, which only requires "more specific", not "perfect".

## Should know
- ADR-2026-021 (accepted 2026-07-12) is the source of truth for "hướng 4" — referenced in code
  comments at both changed call sites, not reproduced here.
- Existing AA-197 tests (`tests/unit/test_aa197_dfs.py`) must keep passing unchanged — they pin
  the 2-arg `build_seed()` fallback behavior that AA-251 must not regress.
