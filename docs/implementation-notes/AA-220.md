# AA-220 — Multi-version export (A) + DOCX (B) + frontend (C) + multi-tour filter (D)

## Decisions
- **Endpoint A path** = `GET /tours/{tour_id}/versions/export?versions=4,5,6&format=csv|xlsx`.
  Registered ABOVE `GET /tours/{tour_id}/versions/{version_num}` (admin_pipeline.py) because
  `version_num` is typed `int` — `"export"` 422s on that route instead of falling through.
- **Reused the get_tour_version_detail SELECT** but `version_num = ANY($2::int[])` for N versions.
  `brand_name` comes from `JOIN shared.tenant_brand_rules` (metadata->>'brand_rule_id'); model = `gc.model_editorial`
  (generated_content has NO brand_name/model_tier column — verified STEP 0).
- **Table orientation:** field × version. Column 0 = `field` (row label), one column per version (`v4`, `v5`…).
  XLSX uses `freeze_panes=(1,1)` so header row + field column stay pinned.
- **Score rows:** included all of score_brand / score_seo / score_structure / score_quality / score_overall
  (the scope wrote "score_quality(overall)" — kept both score_quality sub-score AND score_overall headline to avoid data loss).
- **judge dict keys** (from admin_pipeline.py:153-158): brand_fit / distinct / mission_present / feedback / judge_score
  → mapped to judge_fit / judge_distinct / judge_mission / judge_feedback / judge_score rows.
- `_as_list` copied inline (module convention — no shared helper).

## Stage 2 (DOCX) decisions
- `/mnt/skills/public/docx/SKILL.md` does NOT exist in this environment — built with stock python-docx 1.2.0 (installed, imports OK).
- B's query = get_tour_version_detail SELECT + `tenant_id` (for buyer_market derivation), scalar `version_num = $2`.
  Keeps the LATERAL seo_context join (A dropped it; B needs DFS).
- buyer_market derived inside the conn scope via TenantConfigService + resolve_buyer_market (best-effort try/except).
- Itineraries rendered with `_body()` — splits on `\n`, emits soft line breaks (run.add_break) in ONE paragraph → no flatten.
- keyword_ideas → 5-col table (Keyword/Volume/Competition/Comp.Index/CPC), capped 25 rows, style "Light Grid Accent 1".
- AAA palette: Orange #DB9628 accent, Blackblue #1F2933 headers, Gray #33363D body; off-white #F8F6F2 page bg best-effort (w:background + displayBackgroundShape, wrapped in try/except).
- seo_meta char count surfaced as a `_kv` row.

## Tradeoffs
- Kept the SELECT a trimmed copy of get_tour_version_detail rather than refactoring a shared query —
  the canonical function returns extra DFS/country fields A doesn't need; duplicating avoids touching the live AA-219 endpoint.
