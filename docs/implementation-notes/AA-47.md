## AA-47: Pipeline Stage Inspector

### Decisions
- Added 3 endpoints to `v1_acp.py` (existing router, prefix `/v1/acp`) — not a new router file.
- Route order: `/runs/{run_id}/context` declared BEFORE `/runs/{run_id}` to avoid FastAPI greedy match.
- Gate stage mapping: "s2"=gate1 (stage int 2, after S2), "s3"=gate2 (stage int 3), "s4"=gate3 (stage int 4). Matches `_STAGE_MAP` in `v1_acp_gate.py`. Gate approve/reject calls existing gate router.
- Blog drafts included in `/context` response (added `run_id` filter query) because existing `list_drafts` endpoint has no `run_id` param.
- CIS tours in S1 panel: queried by country (from `acp_runs.country`) via JOIN `raw_tours → generated_content → quality_scores → published_tours`. Latest version only (MAX version_num subquery).
- JSON viewer: inline collapsible tree component, no npm install. Collapsed by default when object >5 keys.
- `s2_aa_tour_matches` JSONB parsed in backend — JOIN to `published_tours` omitted (table schema keeps tour_ids in the jsonb itself, which is sufficient for display).

### Changed
- `v1_acp.py`: added `Query`, `Decimal` imports; 3 new GET endpoints.
- `layout.tsx`: added "Pipeline" nav entry (first in list, maps to `/workspace/pipeline`).
- New files: `pipeline/page.tsx`, `pipeline/[run_id]/page.tsx` in AA-ACP-App.

### Tradeoffs
- Context endpoint queries CIS tables directly (silver/gold_aa_internal) — ok for admin view, acceptable read cost.
- Blog drafts fetched in context endpoint to avoid a 4th frontend fetch call per run.
- Gate 1 (stage 2) inline approve/reject in inspector uses existing `/v1/acp/gate/s2/approve|reject` — same as B2B portal. This means Gate 1 requires a JWT (not admin secret). For admin inspector, Gate 1 inline action only shown if token present.

### Should know
- All ACP tables currently empty — every panel must handle `null` / empty array gracefully.
- `acp_run_context` has no row for a run_id until S2 has started — S0/S1 context shows only CIS tours (which exist by country), S2–S3 context shows empty.
- `shared.acp_runs` (old table) intentionally excluded — only `acp_shared.acp_runs` used.
- `acp_silver_s3.ads_plan` is singular (not `ads_plans`).
